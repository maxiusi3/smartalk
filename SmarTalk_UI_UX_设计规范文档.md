## 1. 设计系统与视觉原则

### 1.1 设计哲学
- **去学习化**: 避免传统教育界面的严肃感，营造轻松娱乐氛围
- **情感化设计**: 每个交互都要传达鼓励和成就感
- **直觉操作**: 减少认知负荷，让用户专注于内容本身
- **沉浸体验**: 创造电影般的视觉体验，增强代入感

### 1.2 色彩系统
```
主色调 (Primary Colors):
- 深蓝色 #1A237E (信任感、专业性)
- 暖橙色 #FF6B35 (活力、温暖、成就感)
- 柔和白 #FAFAFA (简洁、纯净)

辅助色 (Secondary Colors):
- 成功绿 #4CAF50 (正确反馈)
- 中性灰 #757575 (文本、边框)
- 渐变背景 #E8F4FD → #F0F8FF (舒适感)

兴趣主题色:
- 旅行: 天空蓝 #2196F3 + 日落橙 #FF9800
- 电影: 深紫色 #673AB7 + 金色 #FFC107
- 职场: 商务蓝 #1976D2 + 银色 #90A4AE
```

### 1.3 字体系统
```
中文字体:
- 标题: PingFang SC Medium (16-28px)
- 正文: PingFang SC Regular (14-16px)
- 辅助文字: PingFang SC Light (12-14px)

英文字体:
- 标题: SF Pro Display Semibold
- 正文: SF Pro Text Regular
- 代码/标签: SF Mono Regular
```

### 1.4 图标系统
- **风格**: 圆角线性图标，2px线宽
- **尺寸**: 16px, 24px, 32px, 48px
- **颜色**: 单色或渐变填充
- **动画**: 支持微动画过渡

## 7. 交互规范与微动画设计

### 7.1 核心交互原则
```
触摸反馈规范:
- 按钮按下: 0.95倍缩放 + 阴影加深
- 释放动画: 弹性回弹 (spring: 0.8, damping: 0.6)
- 长按反馈: 1.05倍缩放 + 轻微震动
- 禁用状态: 50%透明度 + 无交互反馈

页面转场动画:
- 推入效果: 新页面从右侧滑入 (300ms, ease-out)
- 返回效果: 当前页面向右滑出 (250ms, ease-in)
- 模态弹窗: 从底部滑入 + 背景渐变 (400ms, cubic-bezier)
- 关闭弹窗: 向下滑出 + 背景淡出 (300ms, ease-in)
```

### 7.2 学习进度动画
```
进度条动画:
- 填充效果: 从左到右平滑填充 (800ms, ease-out)
- 颜色变化: 渐变从蓝色到绿色
- 完成闪烁: 3次快速闪烁 + 粒子爆炸效果
- 数字计数: 从当前值到目标值的数字滚动

钥匙解锁动画:
- 阶段1: 钥匙从灰色变为金色 (300ms)
- 阶段2: 360度旋转 + 发光效果 (600ms)
- 阶段3: 勾选标记从中心放大出现 (200ms)
- 音效: 解锁音效 + 触觉反馈

集齐15个钥匙的庆祝动画:
- 所有钥匙同时发光脉冲 (1s)
- 钥匙向中心汇聚形成圆环 (1.5s)
- 金色粒子爆炸效果 (2s)
- 文字"太棒了！"从下方弹入 (500ms)
```

### 7.3 视频播放交互
```
播放控制动画:
- 播放/暂停: 图标平滑变形 (200ms)
- 进度拖拽: 拖拽球放大1.2倍 + 阴影
- 音量调节: 音量条淡入淡出 (300ms)
- 全屏切换: 平滑的尺寸变化 (400ms)

字幕高亮效果:
- 核心词汇: 黄色背景 + 轻微跳动 (每2秒一次)
- 同步高亮: 与音频时间轴精确同步
- 点击反馈: 词汇放大1.1倍 + 定义弹窗

加载状态动画:
- 视频加载: 骨架屏 + 脉冲动画
- 内容加载: 圆形进度条 + 百分比显示
- 网络错误: 友好的错误提示 + 重试按钮
```

### 7.4 情感化微动画
```
成功反馈动画:
- 正确选择: 绿色勾选从小到大弹出
- 星星飞舞: 5颗星星从四周向中心飞舞
- 文字动画: "很棒！"文字带弹性效果出现
- 背景变化: 背景色温暖变化 (2s渐变)

鼓励动画:
- 选择错误: 温和的左右摇摆 (不是红色X)
- 鼓励文字: 从底部温柔滑入
- 图标变化: 错误选项变灰但不消失
- 继续提示: 箭头指向其他选项

加载等待动画:
- 呼吸效果: Logo轻微的放大缩小
- 进度指示: 彩色进度条流动效果
- 文字提示: "正在为你准备精彩内容..."
- 背景动画: 柔和的渐变色移动
```

## 8. 响应式设计与无障碍考量

### 8.1 屏幕适配规范
```
设备支持:
- iPhone SE (375×667): 最小支持尺寸
- iPhone 13 (390×844): 主要测试设备
- iPhone 13 Pro Max (428×926): 大屏优化
- iPad (768×1024): 平板适配（可选）

布局适配策略:
- 安全区域: 适配刘海屏和底部指示器
- 最小触摸目标: 44×44pt (符合Apple HIG)
- 内容缩放: 支持系统字体大小设置
- 横屏模式: 仅视频播放支持横屏
```

### 8.2 无障碍设计
```
VoiceOver支持:
- 所有UI元素都有语义化标签
- 视频内容提供音频描述
- 进度指示器有语音反馈
- 按钮状态变化有语音提示

视觉辅助:
- 色彩对比度: 符合WCAG 2.1 AA标准
- 色盲友好: 不仅依赖颜色传达信息
- 字体大小: 支持放大到200%
- 高对比度: 系统设置自动适配

运动辅助:
- 减少动画: 遵循系统"减少动画"设置
- 替代操作: 支持语音控制
- 停留时间: 长按时间可调节
- 震动反馈: 可在设置中关闭
```

### 8.3 性能优化
```
图像优化:
- 视频预加载: 智能预加载下一个视频片段
- 图片懒加载: 屏幕外图片延迟加载
- 格式选择: WebP格式优先，PNG/JPG降级
- 尺寸适配: 根据设备屏幕密度提供不同尺寸

内存管理:
- 视频缓存: 最多缓存3个视频片段
- 图片缓存: 使用LRU策略清理缓存
- 内存警告: 低内存时自动清理非必要资源
- 后台处理: App进入后台时暂停下载

网络优化:
- 断点续传: 视频下载支持断点续传
- 网络检测: 根据网络状况调整清晰度
- 离线模式: 已下载内容可离线使用
- 数据节省: 蜂窝网络下提供节省模式
```

### 8.4 错误处理与用户反馈
```
错误状态设计:
- 网络错误: 友好的重试界面
- 加载失败: 清晰的错误信息和解决方案
- 内容不可用: 推荐其他可用内容
- 系统错误: 提供用户反馈入口

用户引导:
- 首次使用: 渐进式引导，避免信息过载
- 手势提示: 必要时显示手势操作提示
- 功能发现: 新功能的柔和提示
- 帮助中心: 易于访问的帮助文档

反馈机制:
- 即时反馈: 用户操作立即给出反应
- 进度提示: 长时间操作显示进度
- 成功确认: 重要操作完成后的确认
- 错误恢复: 提供撤销和重试选项
```
